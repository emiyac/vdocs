ceph架构 ​

  • rados 基础存储系统RADOS(reliable, autonomic, distributed object storage) 即可靠的，自动化的，分布式对象存储
  • librados 基础库
  • radosgw 高层应用接口：对象网关接口（对象存储）
  • rbd 高层应用接口：块存储
  • cephfs 高层应用接口：文件系统存储

ceph基本组件 ​

  • osd 用于集群中所有数据的存储，并对其它osd节点发送心跳，向mon提供监控信息
  • mon 即 Ceph Monitor 用于集群管理和监控，维护集群的map信息，维护集群的健康状态，维护集群的PG信息
  • mgr 即 Ceph Manager 用于集群和管理，提供web管理界面，收集集群状态、运行指标、当前性能指标、系统负载
  • mds 要用 cephfs 时才需要配置mds节点，mds 即 Metadata Server 用于提供文件系统的元数据服务，提供文件系统的目录结构，以及文件和目录的权限信息
  • rgw 即 RADOS Gateway 用于提供对象存储服务，对外提供S3和Swift接口

kubernetes中使用rbd ​

⚡不推荐使用这种方式

  • 只有1个副本可以使用rbd,其它的副本会抛MountVolume.WaitForAttach failed for volume "web-page" : rbd image rbd_storage/k8s-volume is still being used
